## 一、消融实验：如何做

### 1.1 ICPO_SOSv4 的三个创新在代码中的位置

打开 `runICPO_SOSv4_mm.m`：

| 创新 | 代码位置 | 行数 | 作用 |
|:--|:--|:--|:--|
| **① SOS 互惠共生** | 第 48-63 行 | `if rand()<expl_ratio` 块内的 SOS 探索 | 替代 CPO 的视觉/听觉 |
| **② 自适应探索率** | 第 49-50 行 | `a_adapt = 2*(0.7*(1-t/T)^0.5+0.3)` | 早期多探索→后期多利用 |
| **③ 停滞触发撤退** | 第 112-150 行 | `if stagnation_counter>=5` 块 | 陷入局部最优时后退 |

### 1.2 四个消融变体

```
ICPO_SOSv4 (Full)
  │  SOS + 自适应探索率 + 停滞撤退
  │
  ├── noAdapt   →  去掉自适应探索率，改回 50% 固定概率
  │               【证明: 自适应探索率的贡献】
  │
  ├── noRetreat →  去掉周期撤退，只保留 SOS + 自适应探索率
  │               【证明: 停滞撤退的贡献】
  │
  ├── noSOS     →  把 SOS 改回 CPO 原版的视觉/听觉策略
  │               【证明: SOS 互惠共生的贡献】
  │
  └── CPO (基线) → 原版 4 策略，不包含任何创新
                   【基准线】
```

### 1.3 每个变体的具体改动

#### noAdapt：去掉自适应探索率

```matlab
% === SOSv4 原始 (第 49-50 行) ===
a_adapt = 2*(0.7*(1-t/MaxIt)^0.5 + 0.3); expl_ratio = a_adapt/2;
if rand() < expl_ratio   % 自适应: 早期~100%, 后期~30%

% === noAdapt 改为 ===
if rand() < 0.5          % 固定 50%, 与 SOSv3 相同
```

**预期**：Mean 从 5223 退化到 ~5808（SOSv3 水平），证明自适应探索率贡献约 10-12%。

---

#### noRetreat：去掉停滞撤退

```matlab
% === SOSv4 原始 (第 112-150 行) ===
if stagnation_counter >= 5
    % ... 整段撤退逻辑 (约 40 行)
end

% === noRetreat 改为 ===
% 直接删除整个 if 块
```

**预期**：Min 值退化（最佳解不再达到 ~4700），均值可能略上升或持平。证明撤退是"偶尔的好解"的来源。

---

#### noSOS：用 CPO 原版探索替代 SOS

```matlab
% === SOSv4 原始 (第 48-63 行) ===
if rand() < expl_ratio  % SOS 互惠共生
    RMV = x_i + rand*(x_rand - x_i)
    x_new = x_i + rand*(gBest - RMV)
end

% === noSOS 改为 (CPO 原版) ===
if rand() < expl_ratio  % CPO 视觉/听觉 (注意保留 expl_ratio!)
    if rand() < rand()  % Strategy 1: Sight
        % ... 正态分布扰动
    else                % Strategy 2: Sound
        % ... 随机差分
    end
end
```

> **关键**：noSOS 仍然保留自适应探索率和停滞撤退。唯一的变化是"探索阶段用什么策略"——SOS vs CPO 原版。

**预期**：这是贡献最大的消融变体。Mean 可能退化到 ~6000+。证明 SOS 互惠共生是核心驱动力。

## 二、对比算法：为什么选这五个

### 2.1 算法矩阵

| 算法 | 粒子数 | 类型 | 选择理由 |
|:--|:--|:--|:--|
| **SPSO** (2021) | 500 | PSO + 球向量 | 🥇 **领域基准**：原论文发表于 *Applied Soft Computing*，所有论文的必比基线 |
| **GWO** (2014) | 150 | 狼群层级 | 🥈 项目已验证：150 粒子可接近 SPSO (Map1 Mean=4924)，证明"不是粒子多就赢" |
| **CPO** (2024) | 150 | 豪猪四防御 | **直接前身**：ICPO_SOSv4 不是凭空产生的，CPO 是基底 |
| **WOA** (2016) | 150 | 鲸鱼包围 | 领域最常用的基线之一（6/10 篇解读论文中都用了），增加对比广度 |
| **ICPO_SOSv4** | 150 | 本文 | 🎯 主角 |

### 2.2 为什么不多加几个（比如 AGWOv5 / ICPO Liu 2024）？

| 候选 | 不加的理由 |
|:--|:--|
| **AGWOv5** | GWO 已在列表中，AGWOv5 是 GWO 的变体。加入后还需解释"为什么 AGWOv5 在本文框架下跑出的结果不同"。审稿人不加分，反而可能质疑你的实现。 |
| **ICPO (Liu 2024)** | ① 代码未开源（高被引 47 但可复现性极低）；② 它用的笛卡尔坐标，你是球坐标——不同搜索空间之间的对比需要在公平条件下重做，工程量翻倍。**可以在 Related Work 中讨论而不加入实验对比。** |

### 2.3 5 个算法就够了

SPSO 论文对比了 7 个算法（PSO + θ-PSO + QPSO + GA + ABC + DE + SPSO），但它的核心对比只有 4 个 PSO 变体 + 3 个其他元启发式。

你这里：**1 个基准 (SPSO) + 1 个同框架 (CPO) + 2 个跨算法 (GWO + WOA) + 1 个本文 = 5 个**，覆盖面与 SPSO 论文相当。

## 三、实验执行顺序

```
Step 1: 创建消融变体 (noAdapt / noRetreat / noSOS)
        从 runICPO_SOSv4_mm.m 复制 → 删改 ~各 10 行

Step 2: 创建 GWO / WOA runners
        从现有 GWO_MAIN.m / WOA_MAIN.m 提取核心循环

Step 3: 编写 batch_ablation.m
        5 变体 × Map1 × 20 次

Step 4: 编写 batch_paper.m
        5 算法 × 4 地图 × 20 次

Step 5: 统计分析
        Wilcoxon (每地图 ICPO vs 各算法) + Friedman (总排名)

Step 6: 画图
        收敛曲线 + 箱线图 + 3D 路径 + 消融柱状图
```
